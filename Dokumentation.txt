Projektdokumentation – PuppyRacer
Kurs: Web-Technologien (WIN), WS 25/26
Entwickler: Jonas Neugebauer
1. Einleitung
PuppyRacer ist eine vollständige E-Commerce Web-Applikation für den Verkauf von Hundezubehör.
Die Anwendung umfasst einen Online-Shop für Kunden sowie einen Admin-Bereich zur Verwaltung von Produkten, Bestellungen und Benutzern.
Live-System:
Frontend: https://htwg-in-schneider.github.io/frontend-puppyracer/
Backend: https://backend-puppyracer.onrender.com

2. Implementierte Features
2.1 Kunden-Funktionen
•	Produktkatalog durchsuchen
•	Warenkorb verwenden
•	Bestellungen aufgeben
•	Bestellbestätigungen ansehen
•	Eigenes Profil bearbeiten
2.2 Admin-Funktionen
•	Produkte verwalten (CRUD)
•	Benutzer verwalten
•	Alle Bestellungen einsehen
•	Bestellstatus ändern

3. Technologie-Stack
Frontend
•	Vue.js 3 mit Composition API
•	Vue Router für Navigation
•	Pinia für State Management
•	Auth0 für Authentifizierung
Backend
•	Spring Boot 3.x mit Java 17
•	Spring Security mit JWT
•	Spring Data JPA / Hibernate
•	H2 Datenbank
Deployment
•	Frontend: GitHub Pages
•	Backend: Render.com

6. Architektur-Übersicht
Frontend-Komponenten
•	HeroSection– Startseite
•	ProductCatalog.vue – Produktübersicht
•	Checkout.vue – Bestellprozess
•	OrderConfirmation.vue – Bestätigung
•	OrderHistory.vue – Bestellübersicht
•	EditProduct.vue – Admin-Editor
Backend-Entities
•	Product – Produktinformationen
•	Order – Bestellungen
•	User – Benutzer mit Auth0
•	OrderItem – Bestellpositionen
API-Endpunkte
•	GET/POST/PUT/DELETE /api/products
•	GET/POST /api/orders
•	GET /api/orders/admin (nur Admin)
•	GET/PUT /api/profile

7. Datenbank-Schema
PRODUCTS: id, name, price, description, category, image, stock, active
ORDERS: id, order_number, user_id, status, total_amount, shipping_address
ORDER_ITEMS: id, order_id, product_id, quantity, price
USERS: id, email, role, auth0_id

8. Besondere Implementierungen
Bestellprozess
•	Warenkorb füllen
•	Checkout mit Adresseingabe
•	POST an /api/orders
•	Backend generiert Bestellnummer (ORD-Timestamp-Zufall)
•	Bestätigungsseite
Bilderverwaltung
Bilder liegen im Verzeichnis /public/product_pics/. In der Datenbank wird ausschließlich der Dateiname gespeichert. Das Frontend generiert den vollständigen Pfad dynamisch.
Responsive Design
•	Mobile-first Ansatz
•	CSS Grid und Flexbox
•	Media Queries für alle Geräte
